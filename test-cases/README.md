# Test Case Documents

This folder stores test case requirements in Markdown format.

## Suggested files

- `homepage.md`
- `product-center.md`
- `contact.md`
- `news.md`
- `about.md`
- `application-note.md`

## Naming convention

- Use domain-based file names.
- Keep one major module per file.
- Use stable project-wide case IDs, e.g. `TC-001`.
- Put classification in metadata: `Feature`, `Area`, `Type`, `Priority`, `Smoke`, and `Tags`.
